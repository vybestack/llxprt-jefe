## What landed

- **Key routing (`src/app_input/prs.rs`)**: 8-level precedence resolver; per-pane
  handlers with Left/Right arrow pane-cycle (RepoList Right→PrCycleFocus; PrList
  Left→Reverse / Right→Forward; PrDetail Left→Reverse, Right unbound);
  Tab/BackTab pane cycling; read-only paths return `Some(PrShowNotice{kind})`
  (ReadOnlyNoComment / ReadOnlyReplyOnComment / ReadOnlyNotEditable) never bare
  None; c/r composer, S chooser, o open/notice; full Esc precedence chain.
- **Input mode (`src/input.rs`)**: `input_mode_for_state` real DashboardPullRequests
  precedence Inline > Chooser > Search > Filter > Normal.
- **Normal-mode wiring (`src/app_input/normal.rs`)**: `handle_dashboard_prs_key`
  runs BEFORE `resolve_mode_key` (p/P in PR mode → RefocusPrList, never a second
  EnterPrsMode); Dashboard-only p/P → EnterPrsMode.
- **Filter controls (`src/app_input/prs_filter.rs`)**: fully interactive — Tab/
  BackTab/Space/text/backspace/Enter/Ctrl-c/Esc; 8-field cycling; Space
  review/checks/draft/state cycling; Apply commits + triggers reload.
- **Dispatch (`src/app_input/mod.rs`)**: `dispatch_prs_message` router.
  - Inline(Submit) arm runs `apply_and_persist(message)` BEFORE
    `prs_mutation::handle_pr_inline_submit` (component-004 line 112); the reducer
    `pr_inline_submit` sets `mutation_pending` + preserves composer text, which
    `resolve_pr_inline_submit` requires to reach `create_pr_comment`.
  - OpenInBrowser arm applies/persists the reducer notice BEFORE the spawn.
  - Send-to-agent: `dispatch_pr_agent_chooser_confirm` reads send-info →
    apply_and_persist(PrAgentChooserConfirm) → write_pr_prompt → launch_pr_agent.
    `launch_pr_agent` / `spawn_and_attach_fresh_for_pr` (ctx==None guard) /
    `persist_pr_agent_launch_success` mirror the issues-mode helpers and reuse
    shared runtime attach/clear helpers.
- **Async loaders (NFR-001)**: all 5 PR gh calls (`list_pull_requests`,
  `get_pull_request_detail`, `list_pr_comments`, `open_pull_request_in_browser`,
  `create_pr_comment`) run inside `spawn_gh_task_with_panic` closures with
  on_panic clearing loading + delivering a `*Failed` event; no UI-thread blocking.
- **Open-in-browser dispatch (`src/app_input/prs_dispatch.rs`)**: NoSelection →
  PrShowNotice(NoSelectionToOpen) no spawn; InvalidSlug → PrOpenInBrowserFailed
  ("Configure repository (owner/name) before opening in browser") — never silent;
  Ok(info) → off-thread spawn.
- **Module split**: comments-page loading extracted into new
  `src/app_input/prs_comments_dispatch.rs` to keep `prs_dispatch.rs` under the
  850-line architecture limit (gate file unchanged; no override added).
- **Viewport**: PR-specific `prs_detail_viewport_rows` added in `src/layout.rs`;
  `update_pr_detail_viewport_rows` calls it (not a raw issues helper).
