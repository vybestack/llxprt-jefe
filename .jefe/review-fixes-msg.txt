Fix pagination lifecycle bugs and add shared ListLoader (#202)

Address review findings from architect/deepthinker on PR #236:

BUG 1 (serious): reload failure no longer destroys the pagination
continuation. begin_reload_with_visibility previously set
next_page = Done before the result arrived; accept_failure only
cleared pending, so any transient reload/silent-refresh failure
silently disabled load-more forever (has_more derives from next_page).
Now next_page only changes on a successful accept_loaded/accept_page;
the pending marker already suppresses load-more during a reload.
Guarded by visible/silent reload-failure regression tests + a PR
silent-refresh integration test.

BUG 2: Actions no-repo path now calls list.clear() (mirrors Issues/PRs)
so a pending reload can't stick forever with no completing task.

BUG 3: Actions repo-navigation now dispatches the list fetch after
reset (mirrors Issues/PRs refresh_repo_scope_if_changed), so the
queued reload actually runs.

  pending matcher deleted; all callers use real monotonic ids.
